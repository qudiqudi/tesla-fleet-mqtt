# Available telemetry fields

Tesla's fleet-telemetry exposes the fields below (from
[`protos/vehicle_data.proto`](https://github.com/teslamotors/fleet-telemetry/blob/main/protos/vehicle_data.proto)).
Add the ones you want to `scripts/register-telemetry.sh` as
`FieldName: {interval_seconds: N}` and re-run it.

Notes:
- Not every field exists on every model/firmware. If you include an unsupported field,
  the `fleet_telemetry_config` call reports it and you simply drop it.
- `interval_seconds` is a hint; many fields publish on change. State lands at
  `<MQTT_TOPIC_BASE>/<VIN>/v/<FieldName>`.
- The bundled config enables a sensible 19-field default (speed, SoC, charging, climate,
  sentry, doors, location, odometer, TPMS).

## Require the `vehicle_location` scope
These need `vehicle_location` granted on the token (see README — use `prompt_missing_scopes=true`):

- DestinationLocation
- DestinationName
- GpsHeading
- GpsState
- Location
- OriginLocation
- RouteLine
